# IncentEdge Beta User Guide

## 5-Minute Quick Start

### Step 1: Create Your First Project
1. Log in at https://incentedge.dev
2. Click "New Project"
3. Fill in:
   - Project name (e.g., "Office Building Solar")
   - Project type (Commercial / Residential / Industrial)
   - Location (State + County)
   - Investment amount (e.g., $500,000)
   - Technology (e.g., Solar, Heat Pump, EV Charging)
4. Click "Create"

### Step 2: Find Eligible Incentives
1. Click on your project
2. Click "Analyze Project"
3. Wait 10-15 seconds
4. See results ranked by eligibility score

### Step 3: Review Incentive Details
1. Click on any incentive to see full details
2. Check eligibility requirements
3. Note application timeline
4. Estimate incentive amount

### Step 4: Export & Share
1. Click "Export Report"
2. Choose format (PDF / Excel / CSV)
3. Share with team or clients

## Searching for Incentives

### Method 1: Keyword Search
Search for program names or types:
- "solar energy tax credit"
- "heat pump rebate"
- "Section 45L new construction"

### Method 2: Semantic Search (AI-Powered)
Describe what you need in natural language:
- "Federal tax credits for energy efficiency retrofits"
- "State rebates for residential solar plus battery storage"
- "Direct pay incentives I can claim immediately"

Results are ranked by relevance, not just keywords.

## Understanding Eligibility Scores

| Score | Meaning | What to Do |
|-------|---------|-----------|
| 95-100% | Perfect match | This project qualifies |
| 80-94% | Strong match | Project likely qualifies |
| 60-79% | Possible match | Review requirements carefully |
| 40-59% | Weak match | Unlikely but possible |
| <40% | Poor match | Unlikely to qualify |

## Testing Tips

### What We Need Most:
1. **Accuracy feedback**
   - Do eligibility matches make sense?
   - Are incentive amounts realistic?
   - Are program descriptions helpful?

2. **UX feedback**
   - What's confusing?
   - What could be easier?
   - Where would you add a feature?

3. **Bug reports**
   - Anything that breaks?
   - Performance issues?
   - Data missing?

### How to Test Effectively:
1. Use real projects when possible
2. Test different states and technologies
3. Try searching in different ways
4. Export reports and review
5. Share with colleagues for feedback

## Getting Help

| Issue | Where to Go |
|-------|-------------|
| Bug report | Post in Slack |
| Feature request | Slack or email |
| Technical help | @aundre on Slack |
| Performance issue | Slack with timing details |

We're here to help!
